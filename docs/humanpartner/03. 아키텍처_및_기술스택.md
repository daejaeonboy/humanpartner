# 아키텍처 및 기술 스택

## 🛠 기술 스택 (Technology Stack)

### 핵심 프레임워크
*   **런타임**: React 19
*   **빌드 도구**: Vite (빠른 HMR, 최적화된 빌드)
*   **언어**: TypeScript (안정성을 위한 엄격한 타입 적용)

### 스타일링 및 UI
*   **프레임워크**: Tailwind CSS (유틸리티 우선)
*   **디자인 시스템**: 자체 구축 "프리미엄 모바일 퍼스트" 디자인 토큰.
*   **아이콘**: Lucide React.
*   **컴포넌트**: 커스텀 컴포넌트 디렉토리 (`/components`), 모듈식 접근 방식으로 마이그레이션 중.

### 백엔드 및 데이터
*   **플랫폼**: Firebase
    *   **인증 (Auth)**: Firebase Authentication.
    *   **데이터베이스**: Firestore (NoSQL).
    *   **호스팅**: Firebase Hosting (`.firebaserc`, `firebase.json` 설정).

### 주요 라이브러리
*   `react-router-dom`: 클라이언트 사이드 라우팅.
*   `axios`: HTTP 요청 (Firebase SDK를 직접 사용하지 않는 외부 API 통신용).
*   `react-datepicker`: 날짜 선택.
*   `@supabase/supabase-js`: (package.json에 명시됨 - 하이브리드 사용 또는 마이그레이션 상태인지 확인 필요).

## 📐 아키텍처 결정 사항

### 1. 모바일 우선 디자인 (Mobile-First Design)
단순히 데스크톱 뷰를 축소하는 것이 아니라, 네이티브 앱과 같은 내비게이션(하단 바)과 터치 친화적인 타겟을 구현하여 모바일 반응성을 최우선으로 고려했습니다.

### 2. 폴더 구조 (Directory Structure)
*   `pages/`: 라우트 컴포넌트.
*   `components/`: 재사용 가능한 UI 블록.
*   `server/`: 백엔드 로직 / Firebase Admin 래퍼 (있는 경우).
*   `docs/`: 프로젝트 문서 (Obsidian Vault).

### 3. 상태 관리 (State Management)
현재는 React Context / Local State에 의존하고 있습니다. 복잡도(사용자 인증, 장바구니, 예약 등)가 증가함에 따라 필요한 경우 전역 스토어(Zustand/Redux) 도입을 고려할 수 있지만, 현재로서는 Context로 충분합니다.
