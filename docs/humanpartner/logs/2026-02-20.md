# 2026-02-20 UI/UX 개선 및 배포 로그

오늘 진행된 모든 수정 사항을 작업 순서대로 정리한 기록입니다.

## 1. 헤더 및 레이아웃 안정화
- **PC 헤더 겹침 해결**: 메인 페이지 상단 헤더가 컨텐츠를 가리는 현상을 수정하기 위해 Spacer 높이를 `175px`로 최적화했습니다.
- **모바일 헤더 최적화**: 모바일 검색창의 높이(Padding)를 확장하고, 우측 아이콘(알림, 메뉴) 크기를 통일하여 균형을 맞췄습니다.
- **검색창 패딩 대칭화**: 검색창 내부 텍스트 시작점(좌)과 아이콘 위치(우)의 간격을 `20px`로 대칭 조정했습니다.
- **헤더 그림자 완화**: 헤더 하단의 그림자 강도를 `shadow-md`에서 한 단계 연한 `shadow-sm`으로 변경하여 부드러운 느낌을 주었습니다.
- **프로필 아이콘 가시성**: 로그아웃 상태일 때 프로필 아이콘이 사라지던 버그를 수정하여 항상 보이게 하고, 클릭 시 로그인 페이지로 연결했습니다.

## 2. 상품 상세 페이지 UI 고도화
- **브랜드 컬러 일괄 적용**: 페이지 전체의 주요 버튼, 뱃지, 활성 탭 라인 등에 메인 컬러(`#FF5B60`)를 일괄 적용했습니다.
- **옵션 탭 디자인 개선**: '추가 구성', '장소', '음식' 탭 영역에 흰색 배경을 입히고 하단 리스트와 여백 없이 결합되도록 수정했습니다.
- **기본 구성 컨테이너 정돈**: 불필요한 테두리를 제거하고 그림자(Shadow)만 남겨 주변 요소와 디자인을 통일했습니다.
- **전체 상단 여백 조정**: 페이지 최상단 여백을 `pt-20`에서 `pt-8`으로 줄여 공간 효율을 높였습니다.

## 3. 예약 달력 및 시스템 로직 수정
- **달력 디자인 정밀 조정**: `react-datepicker`의 불필요한 테두리와 여백을 제거하고, 월 변경 화살표 버튼 위치를 타이틀 높이에 맞춰 정렬했습니다.
- **최소 2일 대여 강제**: 1일만 선택하더라도 자동으로 다음날까지(1박 2일) 선택되도록 `onChange` 로직을 강화했습니다.
- **초기 상태 및 초기화 개선**: 페이지 진입 시나 '일정 초기화' 클릭 시, 오늘부터 내일까지 2일이 기본 선택되도록 수정했습니다.
- **섹션 간격 확장**: 달력 아래 '총 대여 기간'과 '예상 인원' 섹션의 상단 여백을 `pt-8`으로 대폭 넓혀 시각적 답답함을 해소했습니다.

## 4. 용어 및 버튼 텍스트 변경
- **문구 변경 (PC/모바일 공통)**: 
    - "총 결제 금액" → **"예상 견적 비용"**
    - "예약하기" → **"견적 받기"**
- **반영 범위**: 모바일 하단 플로팅 바와 PC 우측 스티키 사이드바 모두에 동일하게 적용했습니다.

## 5. 어드민 배너 및 팝업 시스템 고도화 (DB 스키마 수정)
- **배너 데이터 수정**: 배너 저장 시 `'brand_text'` 컬럼이 없어 발생하는 오류를 해결하기 위해 DB 스키마에 해당 컬럼을 추가했습니다.
- **팝업 시스템 구축**: 이벤트 팝업을 관리할 수 있는 `popups` 테이블을 생성하고, 관리자 페이지에서 팝업 등록 및 수정이 가능하도록 연동했습니다.
- **연결 상품 로직 강화**: 배너와 팝업 모두 특정 상품 코드를 입력하면 해당 상세 페이지로 자동 연결되는 `target_product_code` 필드를 추가했습니다.

## 6. 팝업 클릭 및 네비게이션 개선
- **팝업 클릭 연동**: 팝업 이미지를 클릭했을 때 설정된 링크나 상품 상세 페이지로 정상 이동하도록 로직을 수정했습니다.
- **자동 닫기 기능**: 팝업 클릭 시 페이지가 이동되면서 팝업이 자동으로 닫히도록 개선하여 사용자 편의성을 높였습니다.

## 7. 최종 배포
- **Vite 빌드**: `npm run build`를 통해 최신 코드를 정적 자원으로 변환했습니다.
- **Firebase 배포**: 실서버(Firebase Hosting) 업로드를 완료했습니다.
