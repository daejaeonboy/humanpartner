## 🕒 13:00 - 이메일 인증(회원가입) 기능 구현 및 배포 (Cloud Functions)
**작업 요약**: 회원가입 시 6자리 인증번호를 발송하는 기능을 Firebase Cloud Functions(Serverless)로 구현하고 배포 완료.
**상세 내용**:
1.  **초기 계획 및 변경**:
    - 처음에는 EmailJS(프론트엔드 전용)를 고려했으나, "6자리 인증번호" 보안을 위해 백엔드 전송 방식 채택.
    - 로컬 Express 서버(`server/`)를 검토했으나, 운영 환경(Firebase Hosting)과의 호환성 문제로 **Firebase Cloud Functions**로 전환.

2.  **Firebase Cloud Functions 구현**:
    - `functions/` 디렉토리 생성 및 `functions/src/index.ts` 작성.
    - `nodemailer` 라이브러리를 사용해 Google SMTP 서버 연동.
    - **중요 설정**: Node.js 런타임 버전을 `20`으로 설정 (Node 18 Deprecation 대응).

3.  **환경 변수 및 보안**:
    - SMTP 자격증명(ID/비밀번호)을 코드에서 분리하여 `functions/.env` 파일로 관리.
    - Git 등의 버전 관리에서 제외되도록 처리.

4.  **배포 이슈 해결**:
    - **요금제 이슈**: Cloud Functions (Node 10+) 사용을 위해 Firebase 요금제를 Blaze(종량제)로 업그레이드.
    - **Lint 이슈**: 배포 시 불필요한 스타일 검사 오류가 발생하여 `predeploy` 스크립트 수정.
    - **TS 빌드 이슈**: 외부 라이브러리 타입 충돌 문제 해결을 위해 `skipLibCheck: true` 옵션 적용.

5.  **프론트엔드 연동**:
    - `src/utils/email.ts` 수정: 인증 메일 발송 API를 Cloud Functions URL로 변경.
    - `firebase.json` Rewrite 설정: `/api/email/verify` 요청을 Functions로 자동 라우팅.
