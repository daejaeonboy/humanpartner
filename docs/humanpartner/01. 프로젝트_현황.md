# 프로젝트 현황 및 개요

## 📌 현재 상태
**단계**: 활발한 개발 / 개선 중
**중점 사항**: 사용자 관리, 상품 구성, 모바일 UX

**Humanpartner** 프로젝트(내부명 `partner-mice-booking`)는 MICE(회의, 포상관광, 컨벤션, 전시) 예약 및 파트너 관리를 위해 구축된 웹 애플리케이션입니다. 최근 **프리미엄 모바일 퍼스트** 디자인 언어로 전환했습니다.

## ✅ 최근 완료된 기능
*   **모바일 UI/UX 전면 개편**:
    *   네이티브 앱과 유사한 사용자 경험 구현.
    *   모바일용 하단 내비게이션 바(Bottom Navigation Bar) 추가.
    *   헤더/히어로 섹션을 반응형으로 리팩토링.
    *   프리미엄 디자인 토큰(색상, 타이포그래피) 적용.
*   **상품 관리 (Product Management)**:
    *   **리치 텍스트 에디터**: 상품 설명을 위한 에디터 통합 (`react-quill` 등).
    *   **CMS 관리자 패널**: `ProductManager` 및 공용 `CMSManager`.
    *   **퀵 메뉴**: 상품 카테고리에 퀵 메뉴 아이콘을 동적으로 연결.
*   **백엔드 및 인프라**:
    *   **Firebase 통합**: 인증(Authentication) 및 Firestore 데이터베이스 설정.
    *   **디렉토리 분석**: 웹페이스 및 디렉토리 구조 분석 완료.

## 🚧 진행 중 / 활성 작업
1.  **사용자 관리 (User Management)**:
    *   사용자 프로필 업데이트 기능.
    *   Firebase Auth 처리 (이메일/비밀번호 변경).
    *   프론트엔드와 백엔드 간의 수정 사항 동기화 (Admin SDK).
2.  **상품 구성 고도화 (Product Configuration)**:
    *   복합 상품(예: "음식 구성요소")을 위한 스키마 업데이트.
    *   옵션 및 패키지 관리 기능 개선.
    *   구성요소 선택을 위한 UI/UX 개선.

## 📋 백로그 / 예정 사항
*   **데이터 검증**: 상품 구성 저장 시 엄격한 테스트 수행.
*   **배포**: 자동 배포 규칙 확정 (`/deploy`).
*   **성능**: 에셋 로딩 및 React 렌더링 최적화.
